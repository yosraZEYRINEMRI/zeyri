Innovative Project

OBJECT DETECTION USING MMWAVE RADAR

Context of project

This semester is centered around a large-scale project. This project was in collaboration with a company, and we had to respond to an industrial IoT problem in a multidiciplinary project. The project was proposed by Actia, an international group that specializes in adding value to its customers by providing innovative electronics and diagnostics solutions in several areas such as health, aerospace and defence, automobile and automotive.

I worked with 5 other students which came from very different backgrounds. For that purpose, we had to propose an innovative solution by analysing the problem and then implmenting it after four months.

The purpose of this project is to deliver an embedded system that in the first time, detect obstacles in the near area and give an accurate position (distance, angle, and velocity) on display to prevent the driver from danger; And in the second time, give the drivers the possibility to see what kind of obstacles they are detecting: humans, animals, walls, other vehicles etc.. To do that, we use both camera vision and radar combined to determine position and velocity

Technical Aspects

This project is in collaboration with ACTIA. It concerns manufacturing industries and construction sites where workers can drive different types of vehicles such as cranes, trucks, and more importantly cherry pickers. Sometimes, when driving these vehicles, obstacles happen to be in the blind spot of the drivers and because of inappropriate self-detection equipment, collisions happen. This might cause significant human damages, a delay in production and financial losses. Our goal here is to use the radar technology to detect those obstacles and avoid collisions. More specifically our project mainly focuses on a Texas Instrument (TI) mmWave Sensor radar technology. We tried to study it and use it as a driving-assistance system (ADAS).

The following figure describes the architecture of the system we use:

The project is composed of 7 steps:

  • Make the state-of-the-art radar technology: though ACTIA already provided the radar, we were asked to compare multiple devices and make sure the TI AWR1843 Boost was the appropriate one for this use case.
  • the Understand our radar: After choosing the radar, we made a study to fully understand the main functions, the hardware, block diagram, components, and outputs. We also tried to list out the most important parameters such as range resolution, angle resolution and chirp’s configuration to increase the quality of detection.
  • Experiment: Test the detection for different objects in different conditions. The goal here is to start with Texas Instruments recommended parameters and adjust them depending on the object and the desired application. In one word, make the optimal configuration for our use case.
  • Reverse engineering: to implement inside the code our values. We downloaded the software IDE which includes three software tools (see figure below) :
    Code Composer Studio is an eclipse IDE derivation which enables engineers to program in C language and build the main code. Once the code has been successfully built, we use Uniflash to flash the code onto the ARM microprocessor and visualize the output of the radar on Matlab runtime thanks to the UART serial communication port. We studied the operations in each one of these components to understand how the radar works.
  • Classify obstacles: We added a camera right above the radar and used the Yolov5 python program that we will present inside this report. The purpose here is to combine both obstacles classification and position measurement.
  • Display results: Show to the operator of aerial work platforms in real-time the object detected with a camera alongside the distance and velocity measurement with the radar sensor.
  • Control the whole system from a small 8GB Jetson Tx2 to make it an embedded system and integrate it on a real aerial work platform.

Middleware and Service

Service Oriented Architecture: Room management

I realized lonely a project of Room’s Management to apply Service Oriented Architecture’s and Middleware lessons. The goal was to realize a Web application to manage a room, like opening the door, switching on/off the light depending on the state of a samples of sensors. I had to implement diffrent scenarios based on the state of the sensor. As example of scenario: "open the windows when the temperature is higher than 25°C"

Project overview

For this SOA Project, I decided to work on five different scenarios:

The first scenario, opening / closing of shutter depending on the luminosity: For this service to work properly it was necessary to consider the managing of the shutter, from monitoring the current opening level to updating the shutter’s status.

The aim of the second scenario is to control air temperature. If the temperature is higher than 25°C, windows have to be opened so the room can be ventilated. This scenario uses a temperature sensor and a window actuator.

The third scenario is Air quality scenario: the microservices created was for the CO2 sensor. The CO2 microservice simulates a sensor reading and was fairly simple to code. Accessing the service returns a randomly generated number between 0-3000 ppm. With better planning and time management it would be possible to program this virtual sensor further, adding the more complex functionalities I originally planned for. Unfortunately I had to discard most of these ideas in the interest of finishing the project on time with what I had.

The aim of the fourth scenario is managing closing and opening doors and windows according to constraints of Covid pandemic, here I defined three use cases depending on the number of students in the classroom.

Use case 1: After every use of the classroom and for 15 minutes, the door is automatically closed, and the windows are being opened in order to start the aeration process.

Use case 2: When the number of students in the room is >= 25, then the alarm is trigged until the number of students decreases.

Use case 3: When the number of detected persons is > 0 and < 25, the door is open and the windows are closed:

The fifth scenario, I want to turn on lamp if a presence is detected in the room. To do so, I just need lamp and a presence sensor.

Analog Electronic Labs

After we manufactured the sensor, we had to integrate it in an electronic circuit. The aim of this circuit is to amplify the out signal from the gas sensor in order to be used by an arduino. In fact, the arduino's ADC can not treat the the out signal while this signal is not amplified: the entry impedance of the Arduino is much lower than the impedance of the sensor.

LTSpice: How we simulated the circuit

We used the software LTSpice to simulate this analog circuit. It allows us to build schematics we want and then run some simulations.

The next figure shows the final circuit that we used. Firstly, we simulated the sensor by a low pass filter. Then, we placed it at the entry of the amplifier that can amplify it and obtain the wanted signal. Finally, we had to filter the noise, to do that, we used several filters.

Sensor manufacturing

The Gas Sensor we created is based on tungsten trioxide nanoparticles. Small combs silicon substrates fill with tungsten trioxide nanoparticles (WO3) deposit react to the ambient concentration of gas.

To do that there were three steps:

  • Synthesis of nanoparticles of tungsten

    • We followed a protocol to synthesize tungsten trioxide molecules. It was a very intresting protocol insofar as it allowed us to understand how import the security was in such as chimical experience. Moreover, we understood how import is the ecological impact, and how to reduce it by limiting the waste. Finaly, it was important to be very meticulous in such as experiences.

  • Photolithography: Conception of the combs

    • The purpose of this part is to create the sensor chip which will contain the nanoparticles that we synthesized before. The next figure shows the chip archetecture:

  • DiElectrophoresis: integration of tungsten in the combs

    • For the integration of the nanoparticles into the sensor we used DiElectrophoresis method. This part will allow the resistors to be well electrically connected. In this method, we connected the pins of the comb, then deposit a nanoparticle solution on the sensor. We passed an electric current for one minute. Our particles will, thanks to the dielectrophoretic force, integrate on the comb. Once the DiElectrophoresis done we saw the tungsten particles disposed in the microscope:

Protocols for Wireless Sensor Network

During this Protocols for Wireless Sensor Network courses we discovered protocols and standards used in Wireless Sensor Network. Nowadays, wireless communications are at the center of IoT. Many protocols exist and are designed for different purposes. Some have a long range and a low data rate and other are designed for short range but have a higher data rate. Thus, the purpose was to compare them in order to know which protocol we can use for each system/use case (in terms of cost, range, energy, security).

M2M-5G Protocol

I was in charge of doing a state of the art on the protocol M2M-5G (LTE-M) with three of my classmates.

M2M communication is the system of communication for devices without human involvement. This communication can be directly device to device communication or through a network. For scalability and security, devices need to have connectivity with no human interaction.
